JavaSE_day19：IO流

发表于2024-05-01|更新于2024-05-06
|字数总计:3.5k|阅读时长:13分钟|阅读量:

I/O

IO的分类

  • 根据数据流向分
    • 输入流【读数据】
    • 输出流【写数据】
  • 根据数据类型分
    • 字节流
    • 字符流
输入流 输出流
字节流 字节输入流InputStream 字节输出流OutputStream
字符流 字符输入流Reader 字符输入流Writer

字节流

字节输出流【OutputStream】

java.io.OutputStream

方法：

  • close()：关闭流，释放系统资源
  • flush()：刷新流，强制缓冲的字节被写出
  • write(byte[] b)：将b.length字节写入流中
  • write(byte[] b, int off, int len)：将指定的字节数组写入len字节，从off开始写入
  • write(int b)：将指定的字节写入

当完成流操作时，必须调用close方法，释放系统资源

FileOutputStream类

OutputStream的子类，用于将数据写入文件

构造方法

public FileOutputStream(File file)：创建文件输出流，以指定的File对象表示文件

public FileOutputStream(String name)：创建文件输出流，以指定的名称表示文件

创建一个流对象，必须传入一个文件路径，不存在，会创建文件，存在，会清空文件数据

写出字节数据

  1. 写出字节：write(int b)，每次写出一个字节

  2. 写出字节数组：write(byte[] b)，每次写出数组中的数据

  3. 写出指定长度数组：write(byte[] b,int off,int len)，每次写出，从off索引开始，len字节

数据追加续写

public FileOutputStream(File file, boolean append)

public FileOutputStream(String name, boolean append)

append为true表示追加，false表示清空

写出换行

Windows系统里，换行符号是\r\n

回车符：\r，回到一行的开头

换行符：\n，下一行

字节输入流【InputStream】

可以读取字节信息到内存中

方法：

  • public void close()：关闭资源
  • public abstract int read()：从输入流读取数据的下一个字节
  • public int read(byte[] b)：从输入流读取一些字节，存储到数组b中

FileInputStream类

文件输入流，读取文件中的字节

构造方法

  • FileInputStream(File file)
  • FileInputStream(Stream name)

读取字节数据

  1. 读取字节：read()每次读取一个字节的数据，提升为int类型，读取到文件末尾返回-1

  2. 使用字节数组读取：read(byte[] b)，读取b.length个字节到数组，读到文件末尾返回-1

    使用数组读取，每次读取多个字节，减少了系统间的IO操作次数，从而提高了读写的效率，建议开发中使用。

字节流练习：图片复制

字符流

字节流读取文件时，遇到中文字符，可能显示不完整，因为一个中文占用多个字节。因此字符流出现了，以字符为单位读写数据，专用于处理文本文件

字符输入流【reader】

java.io.Reader表示用于读取字符流所有类的父类，读取字符信息到内存中

方法：

  • public void close()：关闭流，释放资源
  • public int read()：从输入流中读取一个字符
  • public int read(char[] cbuf)：从输入流中读取一些，并存储到字符数组中

FileReader类

java.io.FileReader类读取字符文件

构造方法

FileReader(File file)：创建一个流，给定要读取的File对象

FileReader(String fileName)：创建一个流，给定要读取的文件名称

创建一个流对象时，必须传入一个文件路径

读取字符数据

  1. 读取字符：read()，每次读取一个字符，提升为int类型，读取到文件末尾返回-1

  2. 使用字符数组读取：read(char[] cbuf)，每次读取cbuf的长度字符到数组中，返回读取到的有效字符个数，读取到末尾返回-1

字符输出流【Writer】

方法：

  • void write(int c)：写入单个字符
  • void write(char[] cbuf)：写入字符数组
  • abstract void write(char[] cbuf, int off, int len)：写字符数组的某一部分，off数组的开始索引，len写入的字符个数
  • void write(String str)：写入字符串
  • void write(String str, int off, int len)
  • 写入字符串的某一部分，off字符串的开始索引，len写入的个数
  • void flush()：刷新该流的缓冲
  • void close()：关闭资源

FileWriter类

java.io.FileWriter类是写出字符到文件的便利类。构造时使用系统默认的字符编码和默认字节缓冲区。

构造方法

  • FileWriter(File file)：创建一个新的FileWriter，给定要读取的File对象
  • FileWriter(String fileName)：创建一个新的FileWriter，给定要读取的文件的名称

写出数据

  • 写出单个字符：write(int b)方法每次写出一个字符数据

    1. 参数为int类型四个字节，但是会保留一个字符的信息写出

    2. 未调用close方法，数据只是保存到了缓冲区，并未写入到文件中

关闭和刷新

由于内置缓冲区的原因，不关闭输入流，无法写出字符到文件。关闭流对象，无法写出数据。所以我们既想写出数据，又想继续使用流，需要使用flush方法

  • flush：刷新缓冲区，流对象可以继续使用
  • close：先刷新缓冲区，然后释放资源

写出其他数据

  1. 写出字符数组

    • write(char[] cbuf)
    • write(char[] cbuf, int off, int len)

  2. 写出字符串

    • write(String str)
    • write(String str, int off, int len)

  3. 续写和换行

    字符流只能操作文本文件，不能操作图片、视频等非文本文件

网络编程

软件结构

  • C/S结构：Client/Server结构，指客户端和服务器结构
  • B/S结构：Browser/Server结构，是指浏览器和服务器结构

网络通信协议

  • TCP协议【传输控制协议】
    • 面向连接的，可靠
  • UDP协议【用户数据报协议】
    • 非面向连接的，不可靠
  • 三次握手
    • 第一次，客户端向服务器发送连接请求
    • 第二次，服务器向客户端发出响应，表示接收到了连接请求
    • 第三次，客户端向服务器发送确认信息，确认连接
  • 四次挥手
    • 第一次，客户端向服务器提出结束连接，此时客户端不再发送数据
    • 第二次，服务器接收到释放连接的请求，将最后的数据发送给客户端
    • 第三次，服务器给客户端发送一个释放连接报文
    • 第四次，客户端接收到释放连接报文，回复一个彻底断开的报文

网络编程三要素

  • 协议：网路通信必须遵守的规则
  • IP地址：互联网协议地址
  • 端口号：标识设备进程

InetAddress类

InetAddress类表示IP地址，两个子类：

  • Inet4Address
  • Inet6Address

表示地址：

  • 域名：www.yaorange.com
  • IP地址：202.108.35.210

获取实例：

  • public static InetAddress getLocalHost()
  • public static InetAddress getByName(String host)
  • public static InetAddress getByAddress(byte[] addr)

常用方法：

  • public String getHostAddress() ：返回 IP 地址字符串（以文本表现形式）
  • public String getHostName() ：获取此 IP 地址的主机名

Socket

通信的两端都要有Socket，也叫做套接字，网络通信其实就是Socket间的通信。

Socket分为

  • 流套接字：使用TCP提供可依赖的字节流服务

    • ServerSocket：服务器套接字
    • Socket：客户端套接字

  • 数据报套接字：使用UDP提供尽力而为的数据包服务

    • DatagramSocket：发送接收UDP数据包的套接字
