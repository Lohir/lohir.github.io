Mybatis参数传递总结

多个参数

  • 参数个数较少
    • 方法上：定义多个简单类型参数，在参数前使用@Param("参数别名")注解
    • 语句中：使用#{参数别名}获取参数值
  • 参数个数较多
    • 全部属于某个实体类
      • 方法上：定义单个实例对象
      • 语句中：使用#{对象属性名}获取参数值，实体类中必须有get、set方法
    • 不完全属于某个实体类
      • 方法上：定义单个Map类型对象
      • 语句中：使用#{key名}获取参数值

单个参数

  • 简单类型
    • 方法上：使用String和基本数据类型
    • 语句中：使用#{占位别名}获取参数值
  • POJO 类型
    • 语句中：使用#{属性名}获取参数值
  • Map 集合类型
    • 语句中：使用#{key名}获取参数值
  • List 集合类型/Array 类型
    • 方法上：使用@Param注解指定别名
    • 语句中：使用#{别名}进行遍历

主键返回

使用场景（级联新增）

在一个业务方法中需要同时新增多个表的数据，且表之间存在外键关联

主键自动增长，所以在新增后，将自动增长的值返回给主表的外键字段使用

使用方式

在insert标签中新增keyProperty="实体的主键属性名（不是数据库的字段名）"

Mybatis关联映射

一对一，表关联设计

  • 设计步骤

    • 关系：一对一

    • 关联方式：基于外键关联，外键名关联表名_主键字段名

    • 实际开发中：字段有外键约束，建议后端基于逻辑删除

      逻辑删除：

      1. 在进行删除的表中新增一个字段，标识是否被删除【0/1——0：未删除，1：删除】

      2. 使用update语句更新标识，代替delete删除操作

      3. 逻辑删除后数据依然存在，只是逻辑上被删除了，若要查询需要将标识字段作为条件判断

  • 开发步骤

    • 新增字段，用于标识是否删除
    • 使用update语句实现删除操作
    • 基于逻辑删除的表，在查询与更新时都要默认加上标识字段作为条件判断

一对多，表关联设计

  • 设计步骤
    • 关系：一对多
    • 关联方式：基于外键定义关系，只能从多的表定义外键

多对多，表关联设计

  • 设计步骤
    • 关系：多对多
    • 关联方式：基于中间表
  • 注意：中间表中数据修改思路，一定是先删除再重新新增
