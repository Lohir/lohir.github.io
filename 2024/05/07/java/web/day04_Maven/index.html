JavaWeb_day04：Maven

Maven简单介绍

  • 什么是Maven，Maven能做什么

    Maven是一个构建工具，可以帮我们管理我们的jar包

  • Maven的依赖管理

Maven的使用【重要】

Maven的安装

  • 下载Maven

    因为3.6.0以上版本存在和idea2018的兼容问题，所以本阶段使用3.6.0版本作为项目管理工具

    Apache-maven-3.6.0下载地址：http://archive.apache.org/dist/maven/maven-3/

  • 解压

    目录结构：

    bin:存放了maven的命令，比如我们前面用到的mvn tomcat:run
    boot:存放了一些maven本身的引导程序，如类加载器等
    conf:存放了maven的一些配置文件，如settings.xml文件
    lib:存放了maven本身运行所需的一些jar包

  • 配置环境变量

    MAVEN_HOME：D:\software-for-support\apache-maven-3.6.0

    Path：%MAVEN_HOME%\bin

  • 检查是否安装成功

    mvn -v

Maven配置

  • 创建本地仓库目录

    在根目录新建一个文件夹localRepository

  • 配置本地仓库

    打开conf\settings.xml

    配置本地仓库：<localRepository>本地仓库文件夹的路径</localRepository>

  • 配置镜像仓库

    阿里官方说明文档：https://developer.aliyun.com/mvn/guide

    在settings.xml中找到<mirrors>节点

    id：唯一标识一个镜像
    name：镜像名称
    url：镜像地址
    mirrorOf：代表一个镜像的替代位置，例如central就表示代替官方的中央仓库

IDEA配置

  • 生成Maven项目后，在idea中进入settings，搜索maven【每次新建Maven项目都需要进行配置】
  • 配置Maven home path
  • 配置User Settings file
  • 配置Local repository

插件配置

  • 配置插件

  • Tomcat插件

IDEA打开已有的Maven项目

如果需要将他人的maven项目在自己的idea中打开

  • 删除.idea目录，iml文件，target目录
  • 在idea中直接open项目，如果生成不成功可以重建
